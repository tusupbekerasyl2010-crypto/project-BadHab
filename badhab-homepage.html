BadHab

BadHab is a project designed to help teenagers and students fight bad habits, improve mental health, track moods, and stay motivated through progress analytics and advice systems.

Development Process

Alpha Version

The alpha version focused on the basic idea of the project. Users could select habits, moods, relapse status, and receive advice. The design was simple and mobile-focused.

Sponsor Feedback

The sponsor suggested improving the amount of habits, adding more moods, and making the progress system more interactive.

Watch Alpha Feedback

Beta Version

The beta version improved both design and functionality. A progress analytics graph was added, along with better navigation, more habits, mood tracking, relapse systems, and motivational sections.

Sponsor Feedback

The sponsor appreciated the new graph system, improved structure, and coaching style. However, they recommended adding more interaction and improving the design further.

Watch Beta Feedback
Open Beta Version

Final Version

The final version combines functionality and improved UI. Interactive mood systems now affect the progress graph, users can track relapses, receive motivational quotes, and interact with habit systems dynamically.

Main Improvements

• Interactive graph system
• Mood-based analytics
• Relapse tracker
• Better navigation
• Improved advice system
• Modernized design

Final Reflection

1. What improved the most from Alpha to Final?

The design and functionality have improved significantly. Comparing the two projects, many systems were added and changed, especially the progress analytics graph that was not present in the alpha version.


2. What feedback was most useful?

The most useful feedback came from our sponsor because they paid special attention to the project and evaluated every important detail carefully.


3. What would you still improve?

We would improve the design further by adding more colors, better visual effects, and a more modern interface.


4. Biggest challenge

The biggest challenge was combining design and functionality while keeping the application easy to use.


5. What skill improved the most?

Frontend development and UI/UX design improved the most during this project.


6. What did you learn?

We learned how feedback changes product development, how to improve user experience, and how to develop interactive web applications.
